Factory Inventory Management System

A full-stack demo application for tracking factory inventory, orders, demand forecasts, backlog, and spending. Three-tier architecture with a Vue 3 SPA frontend, a FastAPI backend, and in-memory mock data loaded from JSON files — no database.

Vue 3 Vite 5 FastAPI Pydantic Demo / no database No authentication

01System Architecture

The browser loads the Vue single-page app from the Vite dev server (port 3000). The SPA calls the FastAPI backend (port 8001) over HTTP/JSON; CORS is open for development. The backend serves data held in Python memory, loaded once at startup from JSON files.

03Data Flow

Every page follows the same request cycle, driven by the shared filter state. Four global filters — Time Period, Warehouse, Category, Order Status — apply across views via query parameters.

1
User changes a filter or navigates

FilterBar updates shared refs in the useFilters composable. Views watch these refs and trigger a reload.

2
API client builds the request

api.js converts active filters into query params, skipping any set to 'all', and issues an Axios GET to http://localhost:8001/api/…

3
FastAPI filters in-memory data

apply_filters() matches warehouse/category/status; filter_by_month() handles months (2025-01) and quarters (Q1-2025). Aggregation endpoints compute summaries on the fly.

4
Pydantic validates the response

Results are serialized through typed response models, guaranteeing a consistent JSON shape back to the client.

5
Vue re-renders reactively

Raw responses land in refs (e.g. allOrders, inventoryItems); computed properties derive chart data, totals, and table rows, and the DOM updates.

06Key Design Decisions & Constraints

  • Stateless, read-mostly backend. All data is loaded into memory at startup; filtering is plain Python list comprehensions. Fast for <10K items, no persistence — a restart resets everything.
  • Filters degrade gracefully. The value 'all' (or an omitted param) means "no filter" at every layer — the frontend skips the query param and the backend skips the predicate.
  • Inventory has no time dimension. Only orders carry dates, so the Time Period filter applies to orders and order-derived metrics, never inventory.
  • Business rule: revenue goals. $800K/month for a single month; $9.6M YTD when all months are selected.
  • Frontend/backend can drift. api.js defines methods for tasks and purchase-order creation that have no matching routes in main.py — UI features calling them will 404 until backend endpoints are added.
  • Demo-grade security. No auth, CORS open to all origins, no rate limiting — intended for local development only.
